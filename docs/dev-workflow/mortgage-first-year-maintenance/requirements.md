# 要件定義書: 住宅ローン 初年度維持費の項目別トグル

作成日: 2026-07-21

## 1. 背景・目的

住宅ローンの維持費（管理費・修繕積立金・保険）は現状、経過1年目から満額計上される。しかし実際には、
入居初年度は管理費が日割り・修繕積立金がまだ発生しない・保険は契約タイミングにより初年度に発生しない、
といったケースがある（参考データのシミュレーション表でも1年目の維持費は満額ではない）。

そこで、**初年度（経過1年目）に各維持費を含めるかどうかを項目ごとに選択可能**にし、実態に合わせた
試算をできるようにする。

## 2. スコープ

### やること
- 住宅ローンの維持費3項目それぞれについて、「初年度に含めるか」のトグルを追加する
  - 管理費（初年度に含める / 含めない）
  - 修繕積立金（初年度に含める / 含めない）
  - 保険（初年度に含める / 含めない）
- トグルが OFF の項目は、**経過1年目のその項目の金額を0**にする（2年目以降は変更しない）
- 住宅ローンページ・ライフプラン明細など、維持費を参照する箇所に自動反映する

### やらないこと
- 1年目以外の年（2年目以降）の維持費挙動の変更
- 日割り計算（「含める場合に○割」等の按分）— 本要件は「含める / 含めない」の二択のみ
- 賃貸モードの費用への適用（本件は住宅ローンの維持費に限定）

## 3. 機能要件

| # | 要件 | 優先度 |
|---|---|---|
| 1 | 住宅ローン入力の「維持費・保険」カードに、管理費・修繕積立金・保険それぞれの「初年度に含める」トグルを表示する | 必須 |
| 2 | トグル OFF の項目は経過1年目の金額を0にする（管理費/修繕積立金は複利上昇・段階的の両モードで、保険は1年目の更新分を0にする） | 必須 |
| 3 | 2年目以降の各項目の金額は従来どおり（トグルの影響を受けない） | 必須 |
| 4 | 住宅ローンページの「初年度の月の支払い」「返済スケジュール1年目の維持費・月の支払い」「返済期間中の維持費合計」に反映する | 必須 |
| 5 | ライフプラン明細の1年目「管理費・修繕積立金・保険」に反映する | 必須 |
| 6 | 既定値は全項目「含める（ON）」とし、既存の試算結果を変えない | 必須 |

## 4. 非機能要件

- 互換性: `LifePlanInput` に項目が増えるため、localStorage の保存キーをバージョンアップし、旧データは初期値にフォールバックする（既存の移行方針を踏襲）。
- パフォーマンス: 年次計算量は変わらない（1年目の分岐が増えるのみ）。
- その他: 計算ロジックは `engine/` に閉じ込め、UI から独立してテスト可能に保つ。

## 5. 受け入れ基準

- [ ] 住宅ローン入力「維持費・保険」カードに、管理費・修繕積立金・保険それぞれの「初年度に含める」トグルが表示される
- [ ] 管理費トグルを OFF にすると、経過1年目の管理費が0になり、2年目以降は変化しない
- [ ] 修繕積立金トグルを OFF にすると、経過1年目の修繕積立金が0になり、2年目以降は変化しない
- [ ] 保険トグルを OFF にすると、経過1年目の保険が0になり、次の更新年（6年目など）以降は変化しない
- [ ] 3項目すべて ON（既定）のとき、シミュレーション結果は本機能追加前と一致する
- [ ] 住宅ローンページの初年度の月の支払い・1年目維持費・維持費合計、およびライフプラン明細1年目の維持費に、トグルの結果が反映される
- [ ] 管理費/修繕積立金は複利上昇・段階的の両モードでトグルが機能する
- [ ] エンジンに単体テストが追加され、`npm test` が全て通る
- [ ] `tsc --noEmit` / `eslint` がエラーなし

## 6. 前提・制約

- 対象は `engine/types.ts`（`MortgageConfig`）、`engine/mortgage.ts`（`maintenanceAt`）、`engine/default-input.ts`、
  住宅ローン入力UI（`components/data-input/data-input-page.tsx` の維持費カード）、必要なら `components/mortgage/mortgage-page.tsx`、
  および `engine/__tests__/mortgage.test.ts`。
- 維持費の計算は `engine/mortgage.ts` の `maintenanceAt` に集約されているため、初年度トグルの分岐もここに実装する。
- storage キー（`lib/storage.ts`）を v9 → v10 に上げる。
