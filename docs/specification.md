# MoneyMap 仕様書

最終更新: 2026-07-19  
対象バージョン: 現行実装（`package.json` version `0.1.0`）

## 1. 概要

MoneyMap は、世帯の収入・支出・住宅ローン・投資・ライフイベントを年次で試算し、将来のキャッシュフローと資産推移を可視化するクライアントサイドのライフプランアプリケーションである。

- 表示・入力の単位は原則として **万円**（年額）
- シミュレーションは開始年を 1 年目として、指定年数分を年単位で計算する
- 入力変更時に即座に再計算する
- データはブラウザの `localStorage` に保存する。サーバー、ログイン、他端末との同期はない

## 2. 画面構成

| 画面 | 状態 | 主な内容 |
| --- | --- | --- |
| ホーム | 実装済み | KPI、将来の航路グラフ、平均月額の家計分析、入力画面への導線 |
| データ入力 | 実装済み | 試算に利用するすべての入力値の編集 |
| ライフプラン | 実装済み | 将来の航路グラフ、資産・収支グラフ、年次明細 |
| キャッシュフロー | 実装済み | 資産残高と年間収支のグラフ |
| 資産推移 | 実装済み | 将来の航路グラフ（現金・投信内訳切替可） |
| 住宅ローン | 実装済み | 借入・返済・利息の概況、適用金利の推移、維持費、年次返済スケジュール |
| 目標 / レポート / シミュレーション / 設定 | プレースホルダー | 「準備中」画面を表示。機能は未実装 |

デスクトップでは左サイドバーで画面を切り替える。小画面では画面上部のセレクトボックスで切り替える。単一ページ内で表示状態を切り替える構成であり、画面ごとの URL ルーティングは行わない。

## 3. 入力仕様

データ入力画面は「基本設定」「収入」「支出」「住宅ローン」「ライフイベント」の 5 タブで構成する。数値の入力制約は UI 上ほぼ設けられていないため、妥当な範囲の値を利用者が入力する前提である。

支出に関わるカードは、カード内の入力値から算出した合計（月額合計・年額・全期間合計など）をカード見出しに表示する。

### 3.1 基本設定

| 項目 | 単位 | 内容 |
| --- | --- | --- |
| 開始年 | 西暦 | シミュレーションの 1 年目 |
| シミュレーション年数 | 年 | 算出する年数 |
| 夫・妻の現在年齢 | 歳 | 1 年目の年齢。以後、毎年 1 歳ずつ加算 |
| 子の誕生 | 何年後 | 開始年から何年後に 0 歳となるか。`4` の場合は 5 年目が 0 歳 |
| 夫・妻の現在貯蓄 | 万円 | 現金貯蓄の初期残高。世帯の現金残高として合算 |

### 3.2 収入

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| 夫 | 現在年収 | 1 年目の年収 |
| 夫 | 昇給フェーズ | フェーズごとに昇給額と適用終了の経過年数を設定する。フェーズは追加・削除・名称変更可能 |
| 妻 | 通常時の年収、昇給額、昇給適用年数 | 子育て収入減フェーズに該当しない年の年収を設定する |
| 妻 | 収入減フェーズ | 子の年齢範囲とその期間の年収を設定する。該当期間は通常時年収を上書きする |
| 子ども | 子育て関連手当 | 子の年齢範囲ごとの年額手当を設定する。ステージは追加・削除・編集可能 |

重複する妻の収入減フェーズまたは子育てステージがある場合、配列内で先に登録されたものが適用される。

### 3.3 支出

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| 基礎生活費 | 夫・妻の月額、物価上昇率 | 夫婦の月額を合算し、年額・インフレ込みで計算 |
| 住宅ローン | 借入額、頭金、開始金利、返済期間、ボーナス返済元金 | 通常返済分とボーナス返済分に分けて元利均等の年次返済として計算 |
| 住宅ローン（変動金利） | 金利の上限、見直し間隔、1 回あたりの上昇幅 | 見直しのたびに金利を引き上げ、上限で頭打ちにする |
| 住宅ローン（維持費） | 管理費・修繕積立金（複利上昇 or 段階的）、火災・地震保険、保険更新頻度 | 方式を選択して計算。保険は更新年に一括計上 |
| 固有の支出 | 夫・妻別の 3 フェーズ | 各フェーズの月額とフェーズ 1・2 の終了年を設定 |
| 子ども | 子育て関連費用 | 子の年齢範囲ごとの年額費用を設定 |
| 金融商品積立 | 中期・長期それぞれの初期残高、年額積立、年利 | 各商品を独立して複利計算し、残高を合算 |
| 税金・社会保険料 | 保険料率、基礎控除、住宅ローン控除 | 簡易税計算に使うパラメータ |

### 3.4 ライフイベント

収入イベントと支出イベントを別々に登録する。各イベントは以下を持つ。

| 項目 | 内容 |
| --- | --- |
| イベント名 | 表示用の名称 |
| 年 | 発生する西暦年 |
| 種別 | 収入または支出 |
| 金額 | 正の値で入力する万円単位の金額 |

同じ年・同じ種別のイベントは合算して試算へ反映する。イベントは、対象年がシミュレーション期間外の場合でも保存できるが、試算結果には現れない。

## 4. シミュレーション仕様

### 4.1 年次計算の基本

経過年数を `e`（1 始まり）、開始年を `Y0` とすると、対象年は次式で求める。

```text
year = Y0 + e - 1
夫・妻の年齢 = 現在年齢 + e - 1
子の年齢 = e - 1 - 子の誕生（何年後）
```

子の年齢が 0 未満の場合は未誕生として `-1` を保持し、子育て関連の費用・手当・妻の収入減フェーズは適用しない。

### 4.2 収入

年間の総収入は以下の合計である。

```text
総収入 = 夫の年収 + 妻の年収 + 子育て関連手当 + 収入イベント
```

夫の年収は、昇給フェーズを適用して計算する。各フェーズは終了年まで毎年指定額を積み上げ、最後のフェーズ終了後は年収を維持する。妻の年収は、子の年齢が収入減フェーズに該当する場合はそのフェーズの年収を使用し、それ以外では通常時年収に昇給額を加算する。

```text
妻の通常時年収 = 基準年収 + 昇給額 × min(e - 1, 昇給適用年数)
```

### 4.3 支出

年間の総支出は以下を合計する。住宅ローン控除は支出項目として加算せず、税額の減額に利用する。

```text
総支出 =
  生活費 + 住宅ローン返済額 + 管理費・修繕積立金 + 子育て関連費用
  + 夫の固有支出 + 妻の固有支出
  + 中期積立額 + 長期積立額
  + 税金 + 社会保険料 + 支出イベント
```

生活費は初年度の夫婦月額合計を年額に換算し、2 年目以降は毎年複利で物価上昇率を反映する。

```text
生活費(e) = 月額合計 × 12 × (1 + 物価上昇率 / 100)^(e - 1)
```

固有支出は、該当するフェーズの月額を 12 倍した値である。

### 4.4 住宅ローン

住宅ローンは元利均等返済を **年単位** で計算する。月次返済と繰上返済は扱わない。

借入金額から頭金を引いた額を、**ボーナス返済に充当する元金**とそれ以外（通常返済分）の 2 本に分け、それぞれ独立に償却する。

```text
r = 適用金利 / 100
年間返済額 = 残高 × r / (1 - (1 + r)^(-残返済年数))
利息 = 期首残高 × r
元金返済 = 年間返済額 - 利息
```

適用金利は見直し間隔ごとに上昇幅を加算し、上限に達したらそれ以降は上限を維持する。

```text
適用金利(e) = min(開始金利 + floor((e - 1) / 見直し間隔) × 上昇幅, 上限)
```

金利が変わった年に、その時点の残高・残返済年数・新しい金利で年間返済額を組み直す。最終年は丸め誤差を残さないよう残高を全額精算する。

維持費は管理費・修繕積立金・保険の合計である。管理費と修繕積立金は **複利上昇** と **段階的** の 2 方式から選択でき、管理費と修繕積立金には別々の段階を設定できる。保険は方式によらず、更新年（1 年目、以降は更新頻度ごと）に火災保険と地震保険を一括で計上する。

複利上昇（`compound`）は、初年度の月額から毎年一定率で上昇する。

```text
管理費(e) = 管理費月額 × 12 × (1 + 年間上昇率 / 100)^(e - 1)
修繕積立金(e) = 修繕積立金月額 × 12 × (1 + 年間上昇率 / 100)^(e - 1)
```

段階的（`stepped`）は、金額（月額）と適用終了年（経過年数）の組を並べ、経過年数が収まる最初の段階の月額を使う。長期修繕計画にもとづく段階的な値上げを表現するための方式である。段階は untilYear 昇順で評価し、最後の段階を過ぎた年はその段階の月額を維持する。段階が 1 つも無い場合は 0 とする。

```text
管理費(e) = 段階月額(管理費の段階, e) × 12
修繕積立金(e) = 段階月額(修繕積立金の段階, e) × 12
```

```text
保険(e) = (e - 1) % 更新頻度 == 0 ? 火災保険 + 地震保険 : 0
維持費(e) = 管理費(e) + 修繕積立金(e) + 保険(e)
```

年次シミュレーションでは、通常返済分とボーナス返済分の年間返済額の合計を住宅ローン返済額として、維持費を管理費・修繕積立金・保険として支出に計上する。維持費は完済後も継続して発生する。

年利が 0% のときは、元本を返済年数で均等分割する。返済期間経過後の返済額・残債は 0 となる。

### 4.5 税金・社会保険料

税額は夫・妻を個別に次の順で概算し、合算する。

1. 給与収入から給与所得控除と社会保険料控除、基礎控除を差し引く。
2. 所得税は速算表による累進税率を適用する。
3. 住民税は住民税課税所得の 10% とする。
4. 夫婦の税額合計から住宅ローン控除を差し引く。税額は 0 未満にならない。

社会保険料は夫婦の給与収入合計に設定料率を乗じた概算値である。

```text
社会保険料 = (夫の年収 + 妻の年収) × 社会保険料率
```

住宅ローン控除は 2 年目から設定年数の間に適用する。控除額は、年末残債と対象残高上限の小さい方に控除率を掛けた額である。

```text
控除額 = min(年末残債, 控除対象残高上限) × 控除率 / 100
```

### 4.6 投資・資産残高

年間収支と資産は以下の通り算出する。

```text
年間収支 = 総収入 - 総支出
現金残高(e) = 現金残高(e - 1) + 年間収支
投資残高(e) = 投資残高(e - 1) + 運用益 + 年間積立額
資産合計 = 現金残高 + 中期投資残高 + 長期投資残高
```

投資の運用益は、各年の期首残高に対して年利を乗じる。年間積立額はその年の支出に含めたうえで投資残高に加算するため、資産合計に対して二重計上しない。

## 5. 結果表示仕様

### 5.1 KPI

ホーム画面では以下を表示する。

| KPI | 算出方法 |
| --- | --- |
| 目標達成の確率 | 資産合計が 0 以上かつ年間収支が 0 以上の年数 ÷ 試算年数 × 100（四捨五入） |
| 生涯収支 | 全年の年間収支の合計 |
| 注意すべき時期 | 年間収支が負の連続期間のうち最長の期間 |
| 赤字の可能性がある期間 | 資産合計が負の連続期間のうち最長の期間 |

「目標達成の確率」は統計的な確率計算ではなく、上記の年次状態を割合として表現した指標である。

### 5.2 将来の航路グラフ

資産合計の推移をエリアチャートとして表示する。以下の切替が可能である。

- 横軸: 西暦表示 / 夫の年齢表示
- 系列: 資産合計 / 現金貯蓄と投資信託の内訳

状態は以下の優先順位で判定する。

| 状態 | 条件 | 表現 |
| --- | --- | --- |
| 警告 | 資産合計が 0 未満 | コーラル色 |
| 注意 | 資産合計は 0 以上、かつ年間収支が 0 未満 | アンバー色 |
| 順調 | 上記以外 | グリーン色 |

グラフには開始年を「現在地」、最終年を「目標」として表示し、対象年にイベントがある場合はイベントマーカーを表示する。ホバー時には年、年齢、資産合計、現金貯蓄、投資信託、年間収支、対象年のイベントを表示する。

### 5.3 年次明細と標準グラフ

ライフプラン画面には年次明細を表示する。年を列、収入・支出・残高などを行とする横スクロール可能な表である。行をクリックするとその行をハイライトし、横に長い表でも該当項目を追いやすくする。住宅ローンの残債は住宅ローン画面で扱うため、この明細には含めない。キャッシュフロー画面およびライフプラン画面には、現金貯蓄残高の折れ線グラフと年間収支の棒グラフも表示する。

### 5.4 住宅ローン画面

住宅ローン画面には、借入総額・総返済額・総利息・初年度の月の支払いの概況、適用金利が切り替わる年の一覧、維持費の内訳、および返済期間全体の年次スケジュールを表示する。維持費が段階的な設定のときは、管理費と修繕積立金の合計月額が切り替わる年の一覧もあわせて表示する。スケジュールは「合計」と「内訳（通常・ボーナス）」を切り替えられ、行をクリックするとその年をハイライトする。住まいが「賃貸」の場合は空状態を表示する。

## 6. データ保存

- 保存先: ブラウザの `localStorage`
- 保存キー: `lifeplan-app:input:v3`
- 保存形式: `LifePlanInput` の JSON
- 読込タイミング: 初回クライアント表示後
- 保存タイミング: 初期値以外の入力値が変更されたとき
- JSON の解析に失敗した保存データは無視し、初期値を使用する

初回表示では初期値を表示し、保存済みデータがあればクライアント側で読み込んで置き換える。保存データのスキーマ検証や旧バージョンからの移行は行わない。

## 7. 初期値

初期値は 2026 年開始、64 年間、夫 32 歳・妻 31 歳、4 年後に第一子誕生するシナリオである。夫婦の収入、子育て、住宅ローン、投資および複数のライフイベントが設定済みである。詳細な値は `engine/default-input.ts` を正とする。

## 8. 制約・非対応事項

- 税金・社会保険料は概算であり、扶養控除・配偶者控除・各種税額控除・地域差などは反映しない。
- 住宅ローンは年次元利均等返済のみを扱う。月次計算と繰上返済には非対応。変動金利は「一定間隔で一定幅ずつ上昇し、上限で頭打ちになる」単純モデルであり、実際の金利変動や物価指数との連動は再現しない。
- 夫の退職・再雇用・年金などの収入スケジュールは、個別の昇給フェーズまたはライフイベントで代替する必要がある。
- 子どもは 1 人のみを前提とする。
- 物価上昇は基礎生活費にのみ適用し、その他の支出や収入には自動適用しない。
- 投資は固定年利で毎年運用益を計上する単純モデルで、価格変動、手数料、税金、途中売却・取崩しは扱わない。
- 入力値の範囲・相関関係の検証は限定的であり、負数や期間の逆転など不整合な値を入力しないこと。
- データは端末・ブラウザごとに保存され、エクスポート、インポート、アカウント連携、共有は未実装。
